Hub-and-Spoke Architecture

The Triage Agent serves as the central hub, routing customer requests to specialized spoke agents via handoff mechanisms.

Core Concepts

🔀

Hub Mechanism

The Triage Agent acts as the central hub that receives all incoming messages, analyzes intent, and routes to the appropriate specialist agent. It maintains conversation context and coordinates handoffs between agents.

🔄

Handoff Mechanism

Agents can transfer control to other agents through structured handoff tools. Each handoff preserves the full conversation context, allowing the receiving agent to seamlessly continue the interaction.

🛡️

Input Guardrails

Two guardrails protect the system: Relevance Check ensures queries relate to airline services; Jailbreak Detection blocks attempts to override system instructions or access unauthorized capabilities.
